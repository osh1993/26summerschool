# 수련회 차량 배치

## 데이터 모델
- 이동 요청: `demand_id`, `participant_id`, `direction`, `departure_window`, `pickup`, `dropoff`, `party_size`, `locked_trip`.
- 차량: `vehicle_id`, `label`, `capacity`, `driver_id`, `available_windows`, `route_scope`.
- 운행: `trip_id`, `direction`, `depart_at`, `origin`, `destination`, `vehicle_id`, `passenger_ids`, `status`.

## 절차
1. 이동 요청을 방향·시간창·탑승장소로 묶는다.
2. 수동 고정 운행과 보호자 직접 이동을 먼저 반영한다.
3. 운전자 가능시간과 경로가 맞는 차량만 후보로 둔다.
4. 시간창 위반과 정원 초과를 금지하고, 빈 좌석과 운행 횟수를 줄이는 조합을 찾는다.
5. 미배정 수요와 추가 필요 좌석을 시간대별로 표시한다.
6. 변경 시 영향받는 운행만 다시 계산하고 변경 이력을 남긴다.

## 불변조건
- 실제 탑승 인원은 운전자를 포함한 차량 정원을 넘지 않는다.
- 동일 인원이 겹치는 시간의 운행 두 개에 배정되지 않는다.
- 운전자는 자신의 운행에 명시되고 같은 시간에 다른 운행을 할 수 없다.
- OUT 수요는 해당 시점 이전의 IN 또는 현장 체류 상태와 모순되지 않는다.

## 출력
- 시간표, 차량별 명단, 개인별 이동표, 미배정·정원 경고.
- 공개 데이터에는 전화번호·생년월일·자유서술을 포함하지 않는다.

## 테스트 프롬프트
1. "목요일 본진과 저녁 후발대를 가용 차량 정원에 맞춰 배치해줘."
2. "금요일 저녁 귀가 7명이 추가됐을 때 필요한 차량을 다시 계산해줘."
3. "운전자 한 명이 불참으로 바뀌었으니 영향받는 운행만 재배치해줘."
