# Camp Operations Orchestrator

## 실행 모드: 에이전트 팀

감독자+생성·검증 패턴으로 데이터 설계자, 게시 구현자, QA 검사자가 파일·메시지 기반으로 협업한다. 모든 에이전트 호출은 `model: "opus"`를 명시한다.

## 에이전트 구성
| 팀원 | 역할 | 사용 스킬 | 출력 |
|------|------|-----------|------|
| camp-data-architect | 조편성·차량 데이터와 제약 | camp-group-balancer, camp-vehicle-dispatch | `_workspace/01_data_architect_schema.md` |
| camp-publisher | Google 연동·Pages·Wiki | camp-google-sync, camp-public-dashboard | `apps-script/`, `docs/`, `wiki/` |
| camp-qa | 개인정보·계약·UI 교차검증 | 각 스킬의 불변조건 | `_workspace/04_qa_report.md` |

## Phase 0: 컨텍스트 확인
1. `_workspace/`가 없으면 초기 실행한다.
2. 부분 수정 요청이면 기존 산출물을 읽고 영향받는 에이전트만 재호출한다.
3. 새 Form/엑셀 입력이면 기존 `_workspace/`를 타임스탬프 폴더로 보존하고 새 실행한다.

## Phase 1: 준비
1. 원본 파일·저장소·공개 범위를 감사한다.
2. `_workspace/00_input/`에 파일 경로와 요구사항만 기록하고 개인정보 원문은 복제하지 않는다.
3. 공개 저장소이면 개인정보 비공개 규칙을 최우선 제약으로 등록한다.

## Phase 2: 팀 구성과 작업 등록
1. `camp-data-architect`, `camp-publisher`, `camp-qa`를 `model: "opus"`로 구성한다.
2. 데이터 계약 → 구현 → 점진적 QA 의존성을 등록한다.
3. 데이터 설계와 게시 구조 조사는 병렬 수행하되 공개 JSON 계약을 메시지로 합의한다.

## Phase 3: 설계·구현
1. 데이터 설계자는 조·차량 배정 스키마와 불변조건을 작성한다.
2. 게시 구현자는 Google Apps Script, Pages 대시보드와 Wiki 문서를 만든다.
3. 중간 산출물은 `{phase}_{agent}_{artifact}` 이름으로 `_workspace/`에 보존한다.

## Phase 4: 점진적 QA
1. JSON 생산자와 UI 소비자 필드를 동시에 읽어 비교한다.
2. 정원·중복·시간·미배정 제약을 샘플 데이터로 검사한다.
3. 공개 파일을 개인정보·토큰 패턴으로 검색한다.
4. 모든 링크를 실제 파일 경로와 교차 비교하고 모바일 화면을 검증한다.
5. 결함은 담당 에이전트에게 1회 수정 요청하고 재검증한다.

## Phase 5: 게시
1. GitHub Pages 소스와 Wiki 소스를 별도 커밋 가능 상태로 만든다.
2. 외부 공개 또는 원격 쓰기 전 저장소 공개 범위와 개인정보 검사를 다시 확인한다.
3. Wiki 원격이 비어 있으면 먼저 `Home.md`를 초기화한 뒤 나머지 페이지를 게시한다.

## 데이터 흐름
`Google Form → 응답 시트 → Apps Script 정규화/공개 JSON → GitHub Pages`이며, Wiki는 공시 요약과 Pages 링크를 제공한다.

## 에러 핸들링
| 상황 | 전략 |
|------|------|
| Form 열 매핑 실패 | 원본 보존, SyncLog에 누락 열과 예상 열 기록, 1회 재시도 |
| 배정 제약 충돌 | 미배정으로 남기고 사유 공시, 임의로 제약 완화 금지 |
| Apps Script URL 미설정 | 샘플 데이터 모드로 배포하고 설정 안내 표시 |
| Wiki 초기화 실패 | `wiki/` 산출물과 초기화 명령을 보존하고 Pages 배포는 독립 진행 |
| QA 실패 | 담당자가 1회 수정, 재실패 시 미해결 항목을 명시하고 공개 중단 |

## 테스트 시나리오
### 정상 흐름
Form 신규 응답 → 운영 시트 병합 → 균형 조 제안 → 시간대별 차량 배치 → 비식별 공개 JSON → Pages와 Wiki 갱신.

### 에러 흐름
운전자 불참으로 정원이 부족하면 영향 운행만 재계산하고, 남은 미배정 인원과 필요 좌석을 표시하며 이전 정상 공시본을 유지한다.

## 트리거 검증
### Should trigger
- 여름수련회 조편성을 새로 해줘.
- 금요일 귀가 차량만 다시 배치해줘.
- 구글폼 응답을 웹 공시 화면에 반영해줘.
- Wiki의 조별 표를 업데이트해줘.
- 이전 결과 기반으로 차량 화면을 보완해줘.
- 새 참석자를 넣고 조와 차량을 재계산해줘.
- 부분참석자 이동표를 수정해줘.
- 조편성 점수 가중치를 바꿔 다시 실행해줘.

### Should not trigger
- 안내 가정통신문 문장만 다듬어줘.
- 여행자보험 엑셀의 주민번호 형식을 고쳐줘.
- 수련회 포스터 이미지를 만들어줘.
- GitHub 저장소의 일반적인 브랜치 전략을 설명해줘.
- 교회 버스 사진의 배경을 제거해줘.
- 예산표에서 식비 합계만 계산해줘.
- 설교문 PDF를 요약해줘.
- Google Drive 데스크톱 설치 방법만 알려줘.
