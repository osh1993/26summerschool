# Google Forms/Sheets 동기화

## 원칙
- 원본 Form 응답 시트는 읽기 전용 원장으로 취급한다.
- 운영자가 수정하는 값은 별도 `Participants`, `TravelDemand`, `Vehicles`, `Trips`, `Settings` 시트에 둔다.
- 행 번호 대신 안정적인 ID로 연결한다. 응답 정렬·삽입에도 관계가 깨지지 않기 때문이다.
- 공개 API는 명시적 허용 목록으로 필드를 선택한다.

## 표준 흐름
1. `Settings`에서 Form 열 제목과 표준 필드의 매핑을 읽는다.
2. 타임스탬프 또는 응답 ID로 신규·변경 응답을 병합한다.
3. 운영자 필드(`public_name`, 점수, 고정 조, 확인 상태)는 자동 동기화가 덮어쓰지 않는다.
4. 검증 오류를 `SyncLog`에 남긴다.
5. Apps Script 웹앱의 `doGet`은 공개 허용 필드만 JSON으로 반환한다.
6. 수동 메뉴와 시간 기반 트리거를 모두 제공한다.

## 보안
- 공개 응답에 연락처, 생년월일, 보호자 연락처, 자유서술, 원본 이름을 포함하지 않는다.
- Apps Script URL 외의 비밀값을 저장소에 커밋하지 않는다.
- CORS와 캐시를 고려하되 공개 URL 자체를 인증 수단으로 오해하지 않는다.

2. "공개 API에서 연락처가 절대 나오지 않는지 검사해줘."
